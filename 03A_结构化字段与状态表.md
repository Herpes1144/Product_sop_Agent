# 03A_结构化字段与状态表

## 1. 说明

本文档基于《03_场景建模与产品定义》中的商品质量投诉问题识别与分流处理建议场景整理，用于补充阶段3所需的结构化字段与状态表，作为后续PRD、Axure原型、Coze工作流和字段表整理的基础。

---

## 2. 输入字段表

| 字段名            | 中文名       | 字段说明                                          | 字段类型    | 是否必填 | 来源                | 用途                                 |
| ----------------- | ------------ | ------------------------------------------------- | ----------- | -------- | ------------------- | ------------------------------------ |
| complaint_text    | 客户投诉内容 | 客户本次发起投诉的原始文本内容                    | text        | 是       | 客户输入            | AI识别问题摘要、分类判断的核心输入   |
| customer_id       | 客户ID       | 当前投诉客户的唯一标识                            | string      | 否       | 客户信息系统 / mock | 关联客户信息与历史处理记录           |
| customer_info     | 客户信息     | 客户基础资料，如昵称、账号、标签等                | object      | 否       | 客户信息系统 / mock | 辅助判断客户背景与历史情况           |
| product_id        | 商品ID       | 当前投诉商品的唯一标识                            | string      | 否       | 商品信息系统 / mock | 关联商品详情与规则判断               |
| product_info      | 商品信息     | 商品名称、型号、规格、类目、是否敏感/高风险商品等 | object      | 是       | 商品信息系统 / mock | 支撑质量问题判断与处理路径选择       |
| order_id          | 订单ID       | 当前投诉对应的订单编号                            | string      | 是       | 订单系统 / mock     | 关联订单详情                         |
| order_status      | 订单状态     | 订单当前状态，如已签收、已完成等                  | string      | 是       | 订单系统 / mock     | 辅助判断售后时效与处理条件           |
| receive_time      | 收货时间     | 用户签收或收货时间                                | datetime    | 否       | 订单系统 / mock     | 判断是否在售后时效内                 |
| chat_history      | 历史聊天记录 | 当前客户与售后历史沟通记录                        | text / list | 否       | 聊天窗口 / mock     | 提供上下文辅助判断                   |
| processing_record | 处理记录     | 该工单过往处理过程、备注、状态流转记录            | text / list | 否       | 内部记录 / mock     | 避免重复判断，辅助升级与处理         |
| attachment_list   | 附件材料     | 客户提交的图片、视频、截图等材料                  | list        | 否       | 客户上传            | 辅助判断是否需补材料、是否可继续处理 |

## 3. AI输出字段表

| 字段名               | 中文名           | 字段说明                                                    | 字段类型 | 输出形式 | 用途                                      |
| -------------------- | ---------------- | ----------------------------------------------------------- | -------- | -------- | ----------------------------------------- |
| ai_question_summary  | AI问题摘要       | 对客户投诉内容进行结构化摘要，并包含内容分类                | text     | AI生成   | 帮助一线售后快速理解问题                  |
| problem_type         | 问题分类         | 识别当前投诉属于哪类问题，如破损/瑕疵、功能异常、描述不符等 | string   | AI生成   | 作为后续SOP匹配和动作推荐基础             |
| is_quality_issue     | 是否属于质量问题 | 判断当前投诉是否可归入质量问题范畴                          | boolean  | AI判断   | 影响是否进入质量投诉处理路径              |
| is_in_service_window | 是否在售后时效内 | 判断当前投诉是否仍在可处理时效内                            | boolean  | AI判断   | 影响是否可继续处理                        |
| need_more_materials  | 是否需要补材料   | 判断是否需要客户补充照片、视频、截图等                      | boolean  | AI判断   | 决定是否进入待补材料状态                  |
| can_continue_process | 是否可继续处理   | 判断当前信息是否足够进入下一处理路径                        | boolean  | AI判断   | 决定是否可进入退款/退货退款/换货/补发路径 |
| sop_judgement        | SOP判断依据      | 当前命中的规则、流程、判断依据说明                          | text     | AI生成   | 解释为什么给出当前建议                    |
| next_action          | 下一步动作       | 推荐的一组下一步动作列表                                    | list     | AI生成   | 给一线售后具体执行建议                    |
| recording_summary    | 记录摘要         | 对当前投诉的处理信息进行摘要，方便记录留存                  | text     | AI生成   | 便于内部处理记录与后续追踪                |

## 4. 状态字段表

| 状态值             | 中文名     | 状态说明                                           | 触发条件                          |
| ------------------ | ---------- | -------------------------------------------------- | --------------------------------- |
| pending            | 待处理     | 当前投诉已进入工作台，但尚未完成判断或动作执行     | 新投诉进入 / 尚未处理             |
| waiting_material   | 待补材料   | 当前信息不足，需用户补充照片、视频、截图等材料     | AI判断 need_more_materials = true |
| waiting_escalation | 待升级     | 当前问题复杂或风险较高，需升级处理                 | AI建议升级 / 人工确认升级         |
| resolved           | 已标记处理 | 当前投诉争议已解决，或用户撤回申请，或人工确认关闭 | 人工标记已处理                    |

---

## 5. 动作字段表

| 动作值                 | 中文名                | 动作说明                                 | 触发场景                           |
| ---------------------- | --------------------- | ---------------------------------------- | ---------------------------------- |
| reply_suggestion       | 推荐回复              | 给出当前场景下适合发送给客户的话术建议   | 需要回复客户时                     |
| request_video          | 补视频                | 提示客户补充故障视频                     | 缺少动态证据时                     |
| request_photo          | 补照片                | 提示客户补充商品照片                     | 缺少静态证据时                     |
| request_screenshot     | 补截图                | 提示客户补充订单或聊天截图               | 缺少辅助证明材料时                 |
| escalate               | 升级处理              | 将当前投诉提交给更高层级或专家处理       | 当前问题复杂、风险较高或判断不清时 |
| continue_refund        | 继续处理-退款路径     | 当前规则明确，可继续进入退款处理路径     | 信息充分且适用退款规则             |
| continue_return_refund | 继续处理-退货退款路径 | 当前规则明确，可继续进入退货退款处理路径 | 适用退货退款规则                   |
| continue_exchange      | 继续处理-换货路径     | 当前规则明确，可继续进入换货处理路径     | 适用换货规则                       |
| continue_resend        | 继续处理-补发路径     | 当前规则明确，可继续进入补发处理路径     | 适用补发规则                       |
| mark_resolved          | 标记已处理            | 将当前投诉标记为已处理                   | 用户撤回申请或争议已解决时         |

## 6. 第一版字段范围说明

第一版优先保留以下字段：

### 必须保留
- complaint_text
- product_info
- order_id
- order_status
- ai_question_summary
- problem_type
- sop_judgement
- next_action
- 状态字段
- 动作字段

### 可选保留
- customer_info
- chat_history
- processing_record
- attachment_list
