# 质量投诉分流工作台前端原型

## 项目介绍
本项目是一个面向企业内部一线售后处理人员的前端原型，用于演示“商品质量投诉分流”场景下的工作台操作流程。

当前版本严格聚焦一个核心页面：`质量投诉分流工作台`。页面通过本地 mock 数据展示工单进入、原始投诉信息查看、AI 问题识别结果、SOP 判断依据、建议动作、快捷操作、聊天回复与基础状态切换。

本项目优先面向国内静态托管平台部署，默认发布方式为将 `dist/` 上传至腾讯云 COS / CloudBase 或阿里云 OSS。Vercel 仅作为附加兼容方案保留。

## 本地运行
1. 安装依赖：

```bash
npm install
```

2. 启动开发环境：

```bash
npm run dev
```

## 本地构建
执行下面命令生成纯静态产物：

```bash
npm run build
```

构建完成后，静态产物输出到 `dist/`，可直接用于对象存储静态网站托管。

## 原型范围说明
当前版本只实现以下内容：

- 商品质量投诉分流工作台单页
- 左侧工单展示侧边栏
- 中间上半区原始信息区
- 右侧 Agent 辅助区
- 中间下半区聊天窗口区
- 本地 mock 工单、状态、动作、聊天记录、处理记录
- AI 摘要、分类、SOP 判断依据、建议动作的前端展示
- 快捷操作推荐
- 基础状态展示与切换

当前版本不包含以下内容：

- 登录注册
- 后台管理系统
- 多页面业务流转
- 真实知识库接入
- 真实订单系统接入
- 真实 AI 对话能力
- 自动执行闭环
