# AGENTS.md

## 项目说明
本项目为 **企业售后SOP执行Agent** 的 AI 产品经理项目。
当前第一版严格收敛为：**商品质量投诉分流** 场景下的一线售后 AI 辅助工作台。

## 当前阶段
当前任务不是自由开发，而是基于现有文档继续推进 **Codex 前端原型实现阶段**。
当前主目标是：
- 基于现有 PRD 和前置文档，生成第一版前端可交互原型
- 页面重点为“质量投诉分流工作台”
- 使用 mock 数据完成页面展示与交互验证
- 不得擅自扩展业务边界

## 必须优先阅读的文件
在开始任何实现前，必须先阅读并遵守以下文件：
- `01_项目立项判断.md`
- `02_业务理解与场景分析.md`
- `03_场景建模与产品定义.md`
- `03A_结构化字段与状态表.md`
- `04_PRD.md`

如有结构图附件，也必须参考：
- `项目场景树.pdf`
- `问题类型与SOP结构树.pdf`
- `功能模块树.pdf`

## 第一版范围（必须严格遵守）
第一版只做以下内容：
- 商品质量投诉分流工作台
- 工单展示侧边栏
- 中间原始信息区
- Agent 辅助区
- 聊天窗口区
- mock 商品/订单/聊天/处理记录数据
- AI 摘要、分类、SOP判断依据、建议动作的前端展示
- 快捷操作推荐
- 基础状态展示与切换

## 第一版不做
禁止擅自加入以下内容：
- 登录注册
- 社区模块
- 完整售后工单系统
- 真实企业系统接入
- 真实知识库接入
- 自动审批
- 自动退款/自动退货
- 自动执行闭环
- 其他售后业务场景（退款申请、物流异常、换货/补发独立场景等）
- 权限后台
- 管理后台
- 报表系统

## 产品边界
这是一个 **AI 辅助工作台**，不是聊天机器人，也不是完整后台系统。
AI 在第一版中只做：
- 结构化理解
- 问题分类
- SOP / 规则匹配说明
- 建议动作推荐

AI 不做：
- 最终拍板
- 自动执行操作
- 自动对外承诺

## 页面实现要求
实现时必须遵守以下页面结构：
- 左侧：工单展示侧边栏
- 中间上半区：工单原始信息区
- 右侧：Agent 辅助区
- 中间下半区：聊天窗口区

首屏 P0 信息必须优先保证：
- 客户原始投诉内容
- 商品信息
- 订单信息 / 处理状态
- AI 问题识别结果
- SOP 判断依据 / 建议动作
- 聊天窗口

次级信息：
- 历史聊天记录
- 处理记录

## 数据与状态要求
数据必须以 mock 方式实现，不接真实系统。
必须参考 `03A_结构化字段与状态表.md` 中的字段和状态。

当前统一使用以下状态：
- 待处理
- 待补材料
- 待升级
- 已标记处理

“继续处理”是路径，不是状态。

## 设计与实现原则
- 优先保证结构清晰，不追求复杂炫技
- 优先保证工作台感，而不是聊天感
- 模块层级清楚，避免信息堆叠
- 所有新增内容必须能在现有 PRD 中找到依据
- 如果某个功能在 PRD 中没有明确写出，默认不要擅自增加
- 可以做合理的 UI 优化，但不能改变产品边界

## 输出要求
如果当前任务是生成前端原型，则默认输出应包括：
- 页面代码
- mock 数据
- 基本交互
- 必要注释
- 与 PRD 对齐的模块命名

不要额外创建无关页面。
不要自行补全完整后台。
不要将本项目实现成通用客服系统。

## 冲突处理原则
如果实现冲动与现有文档冲突，始终以以下优先级为准：
1. `04_PRD.md`
2. `03A_结构化字段与状态表.md`
3. `03_场景建模与产品定义.md`
4. `02_业务理解与场景分析.md`
5. `01_项目立项判断.md`

如发现文档冲突，应选择更收敛、更保守、更符合第一版边界的方案。
